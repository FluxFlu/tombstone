Functions

Functions:

A function is a data type that contains code which can be run by "calling" the function. To declare a function, use "fn" For example, 

fn main = () => {
    // Any code included here is a part of the function!
}

The syntax for the function datatype is as follows: `return_type` (...`parameters`) => {`code`} For example, 

// This function accepts an i32 and an i64, and it returns the result of adding them, in the form of an i64. 
fn main = i64 (i32 x, i64 y) => {
    return x + y;
}

A function is called using the syntax: `function_name`(...`parameters`) For example, 

fn main = i64 (i32 x, i64 y) => {
    return x + y;
}
// This function returns 6. 
fn six = i64 () => {
    return main(2, 4);
}

To export functions so they can be run from other files, use the `export` keyword. They can then be imported from other files using "include." 

// file_one.tmb
export fn main = i64 (i32 x, i64 y) => {
    return x + y;
}
// file_two.tmb
#include main from "./file_two.tmb" as main with [i32, i64] returns i64

fn six = i64 () => {
    return main(2, 4);
}

Code is not run directly when executed, you must define a function to be the "entry point" for the code. To do this, use the entry keyword. 

// If you compile this file and run it, `six` is the function that will run.
export fn five = i64 () => {
    return 1 + 4;
}
entry fn six = i64 () => {
    return 2 + 4;
}

Files run this way will be passed a str parameter, containing the arguments the original file was run with. For example, running `node test.js ARGS` will pass `ARGS` to the entry function. If a file that uses the entry keyword is included within a project that doesn't use that file as the entry point, the entry keyword will be treated as though it were the same as the export keyword.
